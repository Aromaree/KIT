# VANVION — оптовый каталог (Telegram Mini App)

Мобильное веб-приложение оптового каталога мужской классической одежды VANVION.
Данные о товарах подгружаются из настраиваемого источника: локального файла `data/products.json`
или опубликованной Google-таблицы в формате CSV.

---

## 1. Как устроены данные

Один товар = одна строка таблицы. Колонки должны называться **ровно так** и идти в этом порядке:

| Колонка      | Назначение                                   | Пример                          |
|--------------|----------------------------------------------|---------------------------------|
| `id`         | Артикул (уникальный)                          | `VN-1001`                       |
| `cat`        | Категория (см. п. 5)                          | `bryuki`                        |
| `name_ru`    | Название (русский)                            | `КЛАССИЧЕСКИЕ ПРЯМЫЕ БРЮКИ`      |
| `name_en`    | Название (английский)                         | `CLASSIC STRAIGHT TROUSERS`     |
| `colors`     | Цвета через запятую (см. п. 4)                | `charcoal,navy,black`           |
| `image`      | Прямая ссылка на фото (см. п. 6), можно пусто | `https://.../photo.jpg`         |
| `comp_ru`    | Состав (русский)                              | `ШЕРСТЬ 100%`                   |
| `comp_en`    | Состав (английский)                           | `WOOL 100%`                     |
| `sil_ru`     | Силуэт (русский)                              | `ПРЯМОЙ`                        |
| `sil_en`     | Силуэт (английский)                           | `STRAIGHT`                      |
| `lin_ru`     | Подкладка (русский)                           | `БЕЗ ПОДКЛАДКИ`                 |
| `lin_en`     | Подкладка (английский)                        | `UNLINED`                       |
| `btn_ru`     | Пуговицы (русский)                            | `КРЮЧОК И ПУГОВИЦА`             |
| `btn_en`     | Пуговицы (английский)                         | `HOOK + BUTTON`                 |
| `country_ru` | Страна (русский)                              | `ИТАЛИЯ`                        |
| `country_en` | Страна (английский)                           | `ITALY`                         |
| `sizes`      | Размерный ряд                                 | `44–58`                         |

Весь текст в каталоге отображается ЗАГЛАВНЫМИ буквами автоматически — можно вводить как удобно.

---

## 2. Настройка Google Таблицы

1. Создайте новую Google Таблицу.
2. В первую строку впишите названия колонок из таблицы выше — **в точно таком же порядке и написании**.
3. Заполняйте по одному товару в каждой следующей строке.

---

## 3. Публикация таблицы как CSV

1. В Google Таблице откройте меню **Файл → Поделиться → Опубликовать в интернете**
   (*File → Share → Publish to web*).
2. В левом списке выберите нужный лист.
3. В правом списке выберите формат **CSV** (значения, разделённые запятыми).
4. Нажмите **Опубликовать** и скопируйте полученную ссылку.
   Ссылка содержит слово `spreadsheets` и заканчивается на `output=csv`.

---

## 4. Как вставить ссылку в приложение

Откройте файл `index.html` и найдите в начале блока `<script>` строку:

```js
<script>
const DATA_URL = './data/products.json';
```

Замените значение на вашу CSV-ссылку:

```js
<script>
const DATA_URL = 'https://docs.google.com/spreadsheets/d/.../pub?output=csv';
```

Приложение само определит формат: если в ссылке есть слово `spreadsheets` — данные читаются как CSV,
иначе — как JSON. Больше ничего менять не нужно.

После добавления новой строки в таблицу достаточно **обновить каталог** — товар появится сразу.

---

## 6. Допустимые ключи цветов (`colors`)

В колонку `colors` пишите ключи через запятую без пробелов, например `charcoal,navy,black`.

| Ключ        | Цвет           |
|-------------|----------------|
| `charcoal`  | Угольный       |
| `navy`      | Тёмно-синий    |
| `black`     | Чёрный         |
| `camel`     | Кэмел          |
| `beige`     | Бежевый        |
| `grey`      | Серый          |
| `brown`     | Коричневый     |
| `green`     | Зелёный        |
| `lightgrey` | Светло-серый   |
| `burgundy`  | Бордовый       |
| `sand`      | Песочный       |
| `slate`     | Сланцевый      |

Неизвестные ключи просто игнорируются.

---

## 7. Как добавить фотографию

Вставьте в колонку `image` **прямую ссылку** на изображение (URL должен заканчиваться на `.jpg`,
`.jpeg`, `.png` или `.webp` и открывать саму картинку, а не страницу с ней).
Если поле `image` пустое — в карточке показывается аккуратная иконка-заглушка.

---

## 8. Возможные проблемы

**Каталог не загружается / ошибка CORS.**
Скорее всего, в `DATA_URL` вставлена ссылка из адресной строки редактора таблицы
(содержит `/edit#gid=...`). Так нельзя.
Используйте только ссылку из меню **Файл → Поделиться → Опубликовать в интернете → CSV**
(она содержит `/pub?output=csv`). Обычная ссылка редактора не отдаёт данные браузеру и вызывает ошибку.

**Изменения в таблице не видны.**
После публикации Google обновляет CSV с небольшой задержкой. Подождите минуту и обновите каталог.

**Не отображается фото.**
Проверьте, что ссылка ведёт прямо на файл изображения, а не на страницу-просмотрщик.

---

## Шрифт

Приложение использует **Futura PT Book**. Положите файл шрифта рядом с `index.html` под именем
`futura-pt-book.ttf`. Если файла нет, автоматически подставляется системный геометрический гротеск
(`Futura`, `Century Gothic`, `sans-serif`).
